\section{Introduction}

\subsection{Motivation}
Undergraduate communications courses introduce a wide span of material---from sampling and quantization to line codes, pulse shaping, passband modulation, and information limits---often in separate homework sets or static figures. It is easy to memorize definitions without building intuition for how parameters (bandwidth, roll-off, noise, symbol rate) change time-domain waveforms, spectra, and eye openings in tandem. This project addresses that gap with a \textbf{single, locally hosted web application} that lets an instructor or student adjust controls and see linked plots update immediately.

\subsection{Objectives}
The team implemented a \textbf{Dash} application (Plotly front end, NumPy/SciPy back end) that:
\begin{itemize}
    \item consolidates ENGR~78 topics into five topical tabs (PCM; line coding; ISI and eye diagrams; $M$-ary PAM and carrier modulation; capacity and simple coding);
    \item runs without specialized hardware in the default configuration;
    \item remains extensible via a clear module layout and an optional ADALM-PLUTO live path for demonstrations.
\end{itemize}

\subsection{Scope and usage}
From the repository root, install dependencies with \texttt{pip install -r requirements.txt}, then run \texttt{python main.py}. Open the URL shown in the terminal in a modern browser; the default address is \path{http://127.0.0.1:8050/}.

The UI uses \texttt{dcc.Tabs}. Each tab exposes sliders, dropdowns, or text inputs that drive \texttt{@app.callback} handlers in \texttt{web\_app.py}. Plots are regenerated server-side and returned as Plotly figure objects.

For optional over-the-air or loopback experiments, run \texttt{python pluto\_live\_app.py} and open \path{http://127.0.0.1:8051/} (separate port from the class visualizer). Use \textbf{Simulation (no radio)} in that UI if hardware is unavailable, then \textbf{Start}. The second Dash application uses modules under \texttt{src/}, including \texttt{realtime\_engine.py}, \texttt{modulation.py}, and \texttt{pluto\_interface.py}. It is \emph{not} required to use or grade the primary course visualizer.

Figures in Section~\ref{sec:results} were exported to \texttt{docs/plot\_guide\_screenshots/} from the running application while preparing documentation; they are representative of the current UI theme and layout.

\section{System description}

\subsection{Software stack}
\begin{itemize}
    \item \textbf{Dash} and \textbf{Plotly} provide layout, routing between tabs, and interactive graphics (pan, zoom, hover) without a separate desktop GUI framework for the main deliverable.
    \item \textbf{NumPy} implements discrete-time signals, FFT-based spectra, convolution for pulse shaping, and Monte Carlo draws for noise and random symbols in the eye-diagram tab.
    \item \textbf{SciPy} supports utilities such as windowed spectral estimates where applicable.
\end{itemize}

\subsection{Functional summary by tab}
\paragraph{PCM.} Users set message bandwidth, oversampling relative to Nyquist, and bits per sample; the interface reports implied sampling rate, quantization levels, and PCM bit rate alongside a conceptual waveform.

\paragraph{Line coding.} A debounced binary string drives six line-code families (on-off and polar NRZ/RZ, AMI, Manchester). The tab shows the voltage waveform and a normalized PSD estimate from a windowed FFT.

\paragraph{ISI and eye.} Raised-cosine roll-off $\beta$, optional neighbor-symbol coupling, additive noise, and PAM order control a random data eye diagram built from overlaid symbol-period segments.

\paragraph{$M$-ary and carrier.} Symbol rate and PAM order illustrate rate--bandwidth tradeoffs. A dropdown selects ASK/OOK, BPSK, BFSK, QPSK, or 16-QAM; the tab shows passband time series, spectrum, and constellation or amplitude views where applicable, plus instantaneous and cumulative throughput.

\paragraph{Capacity and codes.} For a chosen AWGN bandwidth and SNR, the Shannon capacity $C=B\log_2(1+\mathrm{SNR})$ is plotted with the operating point marked. User-specified source probabilities yield entropy $H(X)$. A repetition-code panel illustrates Hamming distance to codewords for a toy three-bit received word.

\section{Discussion and conclusion}

The browser-first design met the practical requirement that every student reproduce the environment on a laptop: installation reduces to a Python virtual environment and \texttt{pip install -r requirements.txt}. Plotly's native interaction (zoom regions of the spectrum, inspect constellation clusters) supports exploratory use during office hours or lab sections.

Limitations are those of any server-side refresh model: many simultaneous callbacks and large FFTs can introduce latency on low-power machines; the Pluto path additionally depends on USB throughput and driver stability. Reasonable defaults in \texttt{web\_app.py} (initial bit patterns, slider ranges) are important so the first screen is pedagogically meaningful without tuning.

Future work could include preset ``lecture modes,'' CSV export of plotted traces, additional modulation or line-code options, or deeper error-control labs---all within the same Dash structure.
